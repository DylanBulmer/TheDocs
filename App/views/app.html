Welcome to The Docs!


This application helps build a knowledge base of issues run into during: the development process, experiments, and other scenarios. To do this, The Docs allows you to publish documents that allows others to find and learn from past experiences.


How to create a document!


On the bottom of the left column, there is a button labeled "Document a Problem!" When you click this button you will pop up with a form that looks identical to a published document. When filling out the form, you will want to make sure the information you are providing is meaningful and will make sense to another user that will be reading your document. In the document you must give: a title that summarizes the issue you have run into, a recommended max is 3-5 words; the project you are working on (if the project is not listed, you can add it in); a description describing the issue you ran into; a URL you found helpful (optional but recommended); some keywords that describe the issues, for example NodeJS, Function, Async this allows the reader to search and understand what your solution is covering; lastly, a detailed solution that clearly describes how to solve the issue.


Search The Docs!


To search, use terms that apply to what you are looking for. This program uses a keyword system to easily find a solution to your issue, but this can only work if you add keywords when producing a document.
